Tailoring the app for our event

In this walkthrough, let’s review the changes you should consider making to the sample before using it for your own event. As an example, we are going to create an app for a Birthday party we hosting for someone who is turning 3! We will take you thru the steps required to submit the app to the Windows Store and send a link to the app to your friends that they can use to download the app to their Windows 10 device with a single click.

Updating the Code

1

The first thing we will do is to update the data models that the app is using. Open EventModel.cs, located in the Models folder of the project.

Tip: Because we have centralized all of the data in one location, making a change ripples thru various parts of the UI that uses the same data bindings. For example, the EventName property is used in the home page of the application (EventMainPage.xaml), and in the code that sends out RSVPs (RSVP.xaml.cs).

2

Change the various properties to the specifics for your event. In this case, we are going to change the EventName from “John’s Band Debut” to “Trisha turns 3!”.

3

In EventDetails.xaml, you will see a Map control we are using(<maps:MapControl/>). If you run the application and navigate to the “Event Details” page, you will see an invalid token message over the map control. This is expected, and let’s fix that.

4

To use the Map control for your own app, you will need to visit the Bing Maps developer portal and request a free token (https://www.bingmapsportal.com/Application) – all you need for this is a Microsoft account.
